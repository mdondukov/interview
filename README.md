## Рекомендуемый порядок прохождения

1. **Языки (Go / Java / Python)** — освежить фундамент, углубиться в рантайм и инструменты.
2. **Алгоритмы** — закрепить базовые паттерны решения задач.
3. **Базы данных** — понять внутреннее устройство и оптимизацию запросов.
4. **Распределённые системы** — изучить теоретические основы (CAP, консенсус).
5. **Системный дизайн** — применить знания на практических кейсах.
6. **Архитектура** — выразить опыт в проектировании микросервисов и интеграций.
7. **AI-интеграция** — разобраться в LLM, RAG и построении AI-систем.
8. **DevOps/SRE** — Kubernetes, CI/CD, observability.
9. **Поведенческие вопросы** — подготовить истории по методу STAR.

Отдельные README внутри каждого трека содержат полный список тем и ссылки на материалы.

## Как работать с репозиторием

1. Выбираешь тему из любого трека → читаешь конспект.
2. Дополняешь своими примерами и заметками.
3. Практикуешься на задачах (для алгоритмов — LeetCode, для system design — mock interviews).
4. Отмечаешь прогресс в соответствующем README трека.
